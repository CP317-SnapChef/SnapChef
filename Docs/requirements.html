1. Introduction

    1.1 Purpose

    The purpose of this document is outline the requirements for SnapChef. The document will describe the features of the app. The intended audience is project team and David Brown. The document will also be useful to our classmates and those who take the class in the future.

    1.2 Scope

    The general aim will be towards those who are unfamiliar with cooking or want to expand their horizons in cooking by finding new uses for different ingredients. Students especially may gain quick cooking experience and understanding through this app. The recipes available will be sorted or restricted by budget and/or servings to allow easier use. Those who have interesting or original recipes, will be able to share on a message board feature.

2. Overall Description

    2.1 Product Perspective

    SnapChef is an app with two major components. There is a frontend interface for the user to interact with, and database containing recipes.

    2.2 Product Functions

    SnapChef is a mobile application. Users input the ingredients they have on hand and mark down any dietary restrictions they may have. They are able to upload their ingredients in three ways. They can input as text, scan the barcode on the packaging or take a picture and let the appâ€™s algorithm determine the ingredients.The app then pulls a recipe matching those ingredients and restrictions.

    2.3 User Characteristics

    Generally, our users will be on a budget and not experienced with cooking, however that does not mean it is strictly available for the inexperienced.

    2.4 Constraints

    • Fake Recipes / Marketing Spam
    • Only for modern android and ios devices
    • Requires access to camera
    • Requires internet connectivity
    • Needs to rely on others for recipes in message boards
    • Cost restraint

    2.5 Assumptions and Dependencies

    • Peer review
    • Responsible imagery
    • Device has camera

    Android Version: 7.0+

    iOS Version: 8.0+

    **depending on teams to decide which version they want to build for**

    3.3 Security

    The SnapChef app is robust against SQL injection. This is accomplished by parameterizing any input that is passed into the SQL statement. As well as opening and closing php connection for querying against the database.

    3.4 Portability

    There is an Android and iOS version. There are shared text input, barcode, ML and recommendation api. If we wanted to expand, then it only needs to create a new frontend.

    3.5 Data Storage

    All of the recipes will be stored in a MySQL Database. MySQL is chosen because of its ability to connect to both IOS and Android applications. There is a single table containing the recipes. The table includes various columns such as Name, Ingredients, Dietary Restrictions etc. We are querying the database through PHP scripts which implement the POST method.

4. Interface Requirements

    We are attempting to make the application on both platforms as similar as possible however due to the differences they experience in both code and hardware, some differences may occur.
