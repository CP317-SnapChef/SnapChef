1. Introduction

It is assumed that all users are either on Android 7.0+ or iPhone iOS 8.0+, as well as a camera along with internet connection. Snapchef is available in the Google Play Store and the Apple App Store.

2. Planned Implementation

    2.1 Choice of Language

    The IOS implementation of SnapChef is built using Swift. IOS development can only happen through Swift or Objective C and our developers have chosen Swift. Similarly the Android implementation of SnapChef is built using Java. Just like IOS, Android development is supported by two languages; Java and Koitlin. Our developers chose Java since its a language they are already familiar with.

    The Database implements NoSQL because of its ability to work with both IOS and Android. The recipes are stored as JSON objects within the database. The database and backend are hosted on AWS Servers.

    The machine learning is being implemented using Tensorflow and the language of Python.

    4.4 Module Interface

    The following are examples of what the final product may look like along with short descriptions of interactive fields from left to right, top to bottom. These are prototypes only and the final product is subject to change

    Login Screen

    This is the screen the user sees when first opening the app or after signing out

    Item name Type Function
    Username input Text box This is where the user enters their username to login
    Password input Text box This is where the user enters their password to login
    Register button Button Brings up the register menu for new users when clicked/tapped

    The taskbar allows the user to switch between different sections of the application. This is displayed at all times for logged in users

5. Maintenence

    5.1 Corrective

    Snapchef will accept feedback in the case that the app crashes. This allows our development team to review the event and correct the bug.

    5.2 Perfective

    The time constraint of this project caused simplification of the product. Snapchef only contains the bare necessary features required in order to ensure the app is operational. New functionality and features are to be developed and implemented once the product is deployed and reliable as well as complete through perfective maintenance.

    5.3 Adaptive

    As Android and iOS platforms are updated to newer versions, Snapchef will be updated to accommodate to the new environment through adaptive maintenance.
